# SEO 점검 요약 (anisb.com)

## 1. 현재 잘 되어 있는 것 ✅

| 항목 | 상태 | 비고 |
|------|------|------|
| **루트 메타** | ✅ | `layout.tsx`: title "아니스비", description, metadataBase, `lang="ko"` |
| **게시글 페이지 메타** | ✅ | `p/[id]`: generateMetadata로 제목·설명·OG·Twitter 카드 |
| **OG 이미지** | ✅ | `opengraph-image.tsx`로 글별 동적 이미지 (제목+본문 일부) |
| **robots.txt** | ✅ | allow/disallow, sitemap URL, 봇별 규칙 |
| **sitemap.xml** | ✅ | 루트·support·게시글 URL, dynamic, created_at 사용 |
| **llms.txt** | ✅ | ANISB 설명·링크·키워드 (영문 스펙 형식) |
| **IndexNow** | ✅ | 새 글 저장 시 URL 제출 (Bing 등) |

## 2. 보완하면 좋은 것 (선택)

- **루트 description**: 현재 "Reddit-like anonymous community" (영문). 한국 검색용으로 한글 설명 추가 또는 병기 (예: "LA 20·30 익명 커뮤니티 아니스비").
- **구조화 데이터**: JSON-LD 없음. 홈에 `WebSite` 또는 글 페이지에 `Article` 스키마 넣으면 검색 결과 풍부해질 수 있음.
- **홈 Open Graph**: 루트는 layout 메타만 있어서 OG가 기본값. 홈 전용 `openGraph`/`twitter` 이미지·설명 지정 시 SNS 공유 시 이득.

## 3. 검색엔진 랭킹과 방문자수 / 트래픽

### 방문자수가 중요한가?

- **직접적인 랭킹 신호는 아님.** Google은 "방문자 수" 자체를 랭킹 요인으로 쓰지 않음.
- **간접적으로는 의미 있음**: 검색에서 들어와서 **체류 시간·이탈률·클릭률(CTR)** 등이 품질 신호로 쓰일 수 있음. 그래서 "자연스러운 검색 유입 → 잘 읽는 페이지"가 중요함.

### 시드 계정으로 방문자수 늘리기

- **효과 거의 없음.** 시드 계정이 같은 IP/기기에서 반복 접속하면:
  - 검색 유입이 아니라 직접/북마크 유입으로만 집계됨.
  - 패턴이 비슷하면 비자연 트래픽으로 분류될 수 있음.
- **랭킹을 올리려는 목적**으로 시드로 "방문 수"만 늘리는 건 비추천.

### 싸게 트래픽·노출 늘리는 **합법적인** 방법

1. **콘텐츠 + 키워드**
   - "LA 한국인 커뮤니티", "아니스비" 등 타깃 검색어에 맞는 글이나 안내 문구를 홈/고객지원 등에 넣기.
   - 새 글이 자주 올라오면 색인·노출 기회가 늘어남 (이미 sitemap·IndexNow 적용됨).

2. **소셜/커뮤니티**
   - 인스타·디스코드·카카오톡 오픈채팅 등에서 **실제 사용자**에게 링크 공유 유도 (무료).
   - "LA 20·30 한국인 모임" 같은 그룹에 서비스 소개 + 링크.

3. **디렉토리·리스트**
   - 해외 한인 커뮤니티 목록, LA 로컬 사이트 디렉토리 등에 **사이트 등록** (무료 또는 소액).
   - 백링크 + 노출 동시에 확보.

4. **로컬/브랜드 검색**
   - "아니스비", "anisb" 검색 시 노출되도록 메타·llms.txt에 이미 반영됨.
   - Google Search Console에서 "아니스비" 등 쿼리 성과 확인 후, 제목/설명 미세 조정.

5. **금지에 가까운 것 (하지 말 것)**
   - **클릭/방문 수를 파는 서비스**: Google 정책 위반·패널티 위험, 비자연 트래픽으로 걸리기 쉬움.
   - **시드로 인위적 방문·체류 시간 만들기**: 검색 랭킹에 실질 이득 없고 리스크만 있음.

---

**요약**: SEO 기본(메타, OG, sitemap, robots, llms.txt, IndexNow)은 잘 되어 있음.  
보완은 한글 설명·구조화 데이터·홈 OG 정도면 충분하고, 랭킹은 **자연 유입 + 좋은 콘텐츠 + 소셜/디렉토리 노출**로 키우는 게 안전하고 효과적임.
